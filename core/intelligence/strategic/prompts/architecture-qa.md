# Architecture Q&A Prompt Template

> 用于 /analyze-architecture 的问答模板

## 问题分类

### 1. 设计原因类

```
模板:
问: 为什么要 {设计点}？
答:
1. {原因1}
   证据: {相关文件/代码}
2. {原因2}
   证据: {相关文件/代码}
3. {原因3}
   证据: {相关文件/代码}

替代方案: {如果有其他常见替代方案}
权衡: {选择当前方案的原因}
```

### 2. 技术选型类

```
模板:
问: 为什么选择 {技术A} 而不是 {技术B}？
答:
{技术A} 的优势:
- {优势1}
- {优势2}

当前项目的选择依据:
- {依据1}
- {依据2}

未来可能的迁移: {如果有}
```

### 3. 模块职责类

```
模板:
问: {模块名} 的职责是什么？
答:
核心职责:
1. {职责1}
2. {职责2}

依赖关系:
├─ 依赖: {被依赖的模块}
└─ 被依赖: {依赖它的模块}

关键类/函数:
- {关键组件1}: {用途}
- {关键组件2}: {用途}
```

### 4. 依赖关系类

```
模板:
问: {组件A} 和 {组件B} 的关系？
答:
关系类型: {依赖/继承/组合/调用等}

{组件A} → {组件B}:
- {具体关系1}
- {具体关系2}

代码位置: {相关代码路径}
```

## 回答格式

```markdown
## 回答

{直接回答问题}

## 理由

1. {理由1}
   - 证据: `{代码片段}` (文件:行号)

2. {理由2}
   - 证据: `{代码片段}` (文件:行号)

## 相关文件

- {文件1}
- {文件2}
```

## 使用建议

1. 每个回答都应包含代码证据
2. 引用具体文件和行号
3. 如果不确定，明确说明"根据代码推断"
