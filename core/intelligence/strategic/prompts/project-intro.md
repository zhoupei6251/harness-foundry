# Project Introduction Prompt Template

> 用于 /understand-project 生成的标准化项目介绍

## 模板

```markdown
# {项目名称}

## 项目概述
{一句话描述项目是什么}

## 技术栈
- **后端**: {后端技术}
- **前端**: {前端技术}
- **数据库**: {数据库}
- **缓存**: {缓存}
- **其他**: {其他基础设施}

## 架构模式
{采用的架构模式，如 MVC、三层架构、微服务等}

## 核心模块
| 模块 | 路径 | 职责 |
|------|------|------|
| {模块名} | {路径} | {职责描述} |

## 设计模式
{使用的设计模式列表}

## 入口点
- API: {API 入口}
- 页面: {页面入口}
- 定时任务: {定时任务入口}

## 关键配置文件
- {配置文件路径}: {用途}
```

## 使用场景

1. **新项目接手**: 快速了解项目全貌
2. **Code Review**: 提供上下文给 Reviewer
3. **多 Agent 协作**: 共享项目上下文
