## 🚫 代码编写

- 🚫 **shell 写文本文件** — 用 `Write`/`Edit` 工具，禁止 `echo >`/`Set-Content`
- 🚫 **未读就写** — 没见过的不写；先 `Read` 相关文件
- 🚫 **静默失败** — 空 catch、返回 null 不抛异常、假装成功

## 🚫 Java / Spring Boot

- 🚫 **Controller 写业务逻辑** — 只做参数校验 + 路由
- 🚫 **循环 SQL** — `for` 循环里发 SQL → 批量
- 🚫 **空 catch** — 必须处理或转译上抛
- 🚫 **泄露敏感信息** — 密码/Token/身份证不打日志
- 🚫 **AOP 自调用** — `this.xxx()` 不走代理，事务/@Cache/@Async 全失效
- 🚫 **JSON 库混用** — 优先级：系统工具 → FastJson2 → Hutool → Jackson

## 🚫 事务与并发

- 🚫 **事务边界错误** — 私有方法调自己不走代理；`@Async` 线程无事务
- 🚫 **事务内缓存更新** — 用 `afterCommit` 回调
- 🚫 **锁还没提交就释放** — 检查 `isHeldByCurrentThread()`
- 🚫 **多实例用本地锁** — 本地 `synchronized` 只锁单 JVM，多实例形同虚设
- 🚫 **分布式锁非原子** — 用 `setnx+expire` 分开 → 用 Redisson 原子命令

## 🚫 安全

- 🚫 **SQL 注入** — `${xxx}` 直接拼接用户输入 → 用 `#{}`
- 🚫 **水平越权** — 改 id 查他人数据 → 所有查询带租户 + 归属校验
- 🚫 **参数篡改** — 后端相信前端价格/金额 → 后端计算
- 🚫 **接口无频率限制** — 注册/登录/验证码 → Redis 限流
- 🚫 **文件上传不校验** — 类型/Magic Number + UUID 重命名

## 🚫 SQL 与性能

- 🚫 **`SELECT *`** — 明确列出需要的字段
- 🚫 **深分页** — `LIMIT 1000000,10` → 游标分页
- 🚫 **循环内发 SQL / HTTP** — 批量改并发
- 🚫 **大 Key / 热 Key 打单节点** — 分片 + 本地缓存

## 🚫 过度设计

- 🚫 **一次调用的抽象** — 直接写，不要 Factory/Strategy/Builder
- 🚫 **"为以后扩展"** — YAGNI：需要时再加
- 🚫 **三层以上嵌套** — 超过 3 层就重构

## 🚫 测试

- 🚫 **修改测试让代码通过**
- 🚫 **跳过测试验证** — 必须实际运行

## 🚫 Git / 部署

- 🚫 **自动 push** — 永远不自动 push
- 🚫 **未经审查合并** — 必须 Code Review 通过

## 过度设计检测

**遇到触发词，停下来自问：**

| 触发词 | 三个问题 |
|--------|----------|
| `Factory`、`Strategy`、`Builder`、`Abstract` | 1. 有 2 个以上调用方吗？ |
| 多层接口 + 实现 | 2. 这个工厂被调用 2 次以上吗？ |
| `"为以后扩展"` | 3. 真的有被扩展需求吗？ |

**答案全是"否" → 直接写，不要抽象。**
