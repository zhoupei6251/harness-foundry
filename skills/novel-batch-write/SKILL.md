# Novel Batch Write — 批量写作

> 当用户说"写到第N章"时使用此模式。
> **智能调度并行/串行写作，最大化效率。**

## 触发词

| 触发 | 说明 |
|------|------|
| 写到第X章 | 批量写作到指定章节 |
| 批量写 | 批量写作模式 |
| 连续写 | 连续多章写作 |

---

```
┌─────────────────────────────────────────────────────────┐

│                                                          │

│  - 当前章节：第{start}章                               │
```

## 章节依赖分析

### 可并行章节

```markdown
章节依赖分析：

| 章节 | 前置条件 | 可并行组 |
|------|----------|----------|
| 第1章 | 无 | A组 |
| 第2章 | 无（独立情节） | A组 |
| 第3章 | 无（独立情节） | A组 |
| 第4章 | 无（独立情节） | A组 |
| 第5章 | 无（独立情节） | A组 |

→ 可并行：第1-5章
```

### 需串行章节

```markdown
| 章节 | 前置条件 | 说明 |
|------|----------|------|
| 第6章 | 依赖第5章结局 | 必须先完成第5章 |
| 第7章 | 依赖第6章 | 必须先完成第6章 |
| 第8章 | 依赖第7章 | 必须先完成第7章 |

→ 串行：第6-8章
```

---

## 执行策略

### Phase 1: 并行写作（≤5个）

```markdown
并行派发：

WU-1: 写第1章 | agent: writer | 独立
WU-2: 写第2章 | agent: writer | 独立
WU-3: 写第3章 | agent: writer | 独立
WU-4: 写第4章 | agent: writer | 独立
WU-5: 写第5章 | agent: writer | 独立

并行度：5
预计时间：~10分钟
```

### Phase 2: 串行写作

```markdown
串行执行：

WU-6: 写第6章
WU-7: 写第7章
WU-8: 写第8章

预计时间：~15分钟
```

## 异常处理

### 字数不足

```markdown
⚠️ 第{XXX}章字数不足

检测到：{字数}字 < 2000字（要求）

选项：
1. 继续扩写至2000字
2. 标记为草稿，稍后完善
3. 跳过，继续写下一章

请选择 [1/2/3]：
```

### 情节冲突

```markdown
⚠️ 检测到情节冲突

第{XXX}章与第{YYY}章存在冲突：
- 第{XXX}章：{冲突内容}
- 第{YYY}章：{冲突内容}

选项：
1. 自动修正（保持一致性）
2. 人工确认（暂停等待指令）
3. 忽略（继续写作）

请选择 [1/2/3]：
```

## 决策树

```
用户输入
    ↓
┌─────────────────────────────────────────┐
│  分析章节依赖关系                        │
├─────────────────────────────────────────┤
│  IF 全部独立 → 全并行（≤5个一批）      │
│  IF 部分依赖 → 并行 + 串行组合          │
│  IF 全依赖 → 纯串行                     │
│  IF 章节数>10 → 分批执行               │
└─────────────────────────────────────────┘
    ↓
执行计划
```
