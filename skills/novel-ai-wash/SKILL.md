# Novel AI Wash — 深度文风清洗引擎

> 比 `humanizer-zh` 更激进的多轮清洗体系。借鉴 ainovel-cli 的 Anti-AI tone 检查和 webnovel-writer 的 15 类 AI-tic 检测。
> **与 humanizer-zh 分工：** humanizer-zh 做单章轻量润色；novel-ai-wash 做批量深度清洗。

## 激活条件

- 用户要求深度清洗（"批量去AI味"、"深度润色"）
- 多章需要统一清洗（≥3 章）
- humanizer-zh 清洗后仍有明显 AI 痕迹
- 准备交付前的最终质量控制

## 四层清洗体系

### Layer 1: 词级替换 + 语义判据

目标：消除 AI 高频词汇和套话，并对可疑句子做语义级 AI 味判断

#### 1.1 黑名单词替换

| 类型 | 替换策略 | 来源 |
|------|---------|------|
| 连接词 | "首先/其次/最后" → 自然因果过渡 | traps #1 |
| 套话过渡 | "就在这时" → 动作/环境变化 | traps #4 |
| 万能结尾 | "预知后事如何" → 具体悬念钩子 | traps #5 |
| AI 高频动词 | "进行"、"展开"、"呈现" → 具体动词 | 统计词频替换 |
| 虚化修饰 | "似乎"、"仿佛" → 直接陈述 | traps #15 |

#### 1.2 语义判据（借鉴 ainovel-cli 语义判据）

**不只看词，还要判断"这句话读起来有没有 AI 味"。** 对每句经过 1.1 替换后的文本，逐句过以下判据：

| 判据 | 检测内容 | 标记 |
|------|---------|------|
| **均匀信息密度** | 连续 5 句每句信息量基本均等（无重点句、无省略跳跃） | ⚠️ AI 味 |
| **情绪扁平化** | 全文情绪曲线接近直线 — 检查情绪词变化频率 | ⚠️ AI 味 |
| **连接词密度** | 每 200 字含 ≥ 3 个逻辑连接词（"因此"、"然而"、"此外"、"与此同时"） | ⚠️ AI 味 |
| **概括句频率** | 段落结尾频繁出现总结性概括句（"总而言之"、"这说明了"、"由此可见"） | ⚠️ AI 味 |
| **形容词均匀分布** | 每段形容词数量大致相等（人类会有的段密集修饰、有的段干脆利落） | ⚠️ AI 味 |
| **万能解释句** | 人物内心独白出现"他知道，这一切都是因为…"式的全知解释 | ⚠️ AI 味 |

**判定流程：**
1. 逐句过语义判据
2. 命中 ≥ 2 项 → 标记为 "⚠️ AI 味"
3. 标记句子进入 L1 重写队列
4. 重写后重新过判据 → 不过则继续标记 | 通过则放行

### Layer 2: 句式重构

目标：打破 AI 均匀分段和机械节奏

- **长短句交错**：连续 3 句超过 30 字 → 插入短句
- **打破三段式**："既…又…还…" → 拆分或直接陈述（traps #11）
- **减少破折号**：每 500 字破折号 ≤ 1 个（traps #12）
- **否定排比重写**："不是A，不是B，而是C" → "是C"（traps #13）
- **连接短语剥离**："不仅如此"、"更令人惊讶的是" → 删除（traps #14）

### Layer 3: 叙事重写

目标：从 AI 的 "分段逻辑" 转为 "人类叙事节奏"

- **信息密度调整**：AI 倾向每段信息量均匀 → 制造疏密节奏
- **场景过渡自然化**：AI 倾向场景切换不带过渡 → 用环境/时间线索衔接（traps #26）
- **对话+动作交错**：AI 倾向纯对话或纯叙述 → 融合对话/动作/心理（traps #27, #39）
- **情绪递进**：AI 倾向情绪平直 → 添加起伏和层次（traps #6）

### Layer 4: 人物声音分化

目标：消除 AI 导致的角色声音趋同

- **每个角色建立语言指纹**：常用词、句式偏好、口头禅
- **对话风格差异化**：身份 × 性格 × 场景 → 不同的说话方式
- **检查人物一致性**：对照 `MEMORY.md` 中的人物设定

## 批量模式

支持多章并行清洗（≤5 章/批）：

```
批次输入: 第1-5章
  ├─ 逐章 L1 词级替换
  ├─ 逐章 L2 句式重构
  ├─ 逐章 L3 叙事重写
  ├─ 逐章 L4 人物声音分化
  └─ 跨章一致性检查（同一人物在不同章中的声音是否一致）
```

## 质量检查

清洗后必须执行：

1. **朗读检查**：读出来是否有 "AI 味"
2. **剥离检查**：移除润色后内容是否空洞（如果是 → 说明原文就空）
3. **对比检查**：与人类作者作品段落对比，判断自然度
4. **人物声音检查**：不同角色对话是否能区分说话者

## 与 humanizer-zh 的分工

| | humanizer-zh | novel-ai-wash |
|---|---|---|
| **适用场景** | 单章轻量润色 | 批量深度清洗 |
| **清洗深度** | 浅层（词级 + 句式） | 深层（四层全覆盖） |
| **速度** | 快（单章 ~1 轮） | 慢（单章 ~3 轮） |
| **适用时机** | 每章写完后 | 整卷完成后 / 交付前 |
| **协作** | 先 humanizer-zh 快速去味 | 如仍有痕迹 → novel-ai-wash 深度处理 |

## 禁止

- ❌ 改变情节走向/增删内容
- ❌ 保留明显 AI 套路化表达（即使只是 "轻微"）
- ❌ 清洗后文本比原文更不自然（过度清洗）
- ❌ 跳过质量检查直接交付
