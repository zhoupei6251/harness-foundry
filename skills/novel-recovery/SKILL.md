# Novel Recovery — 会话恢复

> 当用户重新打开会话或说"继续写作"时使用。
> **快速恢复上下文，继续上次的进度。**

## 触发词

| 触发 | 说明 |
|------|------|
| 继续 | 继续上次进度 |
| 接着上次 | 恢复上次会话 |
| 上次写到 | 显示上次进度 |
| 恢复 | 恢复会话 |

---

## 恢复流程

```
1. 检查 MEMORY.md 状态
       ↓
2. 显示恢复摘要
       ↓
3. 确认继续写作
       ↓
4. 加载上下文
       ↓
5. 继续写作
```

## 快速恢复 vs 完整恢复

### 快速恢复（≤5分钟）

```
用户: "继续"
         ↓
┌─────────────────────────────────────────┐
│  恢复摘要：                              │
│  - 上次写完第5章                        │
│  - 下一步：写第6章                      │
│                                         │
│  ✓ 继续写第6章？                        │
└─────────────────────────────────────────┘
         ↓
用户: "好"
         ↓
加载 MEMORY.md → 写第6章
```

### 完整恢复（>5分钟）

```
用户: "继续"
         ↓
┌─────────────────────────────────────────┐
│  完整恢复检查：                          │
│                                         │
│  📚 书籍状态                            │
│  - 书名：《{书名}》                    │
│  - 章节：1-5章完成                     │
│  - 人物：3个登场                       │
│  - 伏笔：2个埋设                        │
│                                         │
│  📝 待推进情节                          │
│  - 主角需要做出选择A或B                 │
│  - 伏笔X待回收                          │
│                                         │
│  🔍 检测到可能的上下文丢失               │
│  - 建议回顾第5章结尾                    │
│                                         │
│  [快速继续]  [回顾上下文]  [重新规划]   │
└─────────────────────────────────────────┘
```

## 上下文加载

### 最小上下文

```markdown
从 MEMORY.md 加载：
- book_title
- last_chapter
- current_arc
- active_characters
- unresolved_plot_devices
```

### 扩展上下文（如需）

```markdown
额外加载：
- 第N章正文（当章节有续写依赖时）
- 人物设定详情（当涉及复杂人物互动时）
- 大纲当前段落（当需要回顾情节走向时）
```

---

## 中断检测

```markdown
检测中断类型：

| 类型 | 检测条件 | 处理方式 |
|------|----------|----------|
| 正常完成 | 上章已交付 | 直接继续下章 |
| 章节中断 | 字数<2000 | 继续写完 |
| 审稿中断 | 有未完成审稿 | 继续审稿 |
| 润色中断 | 有待润色章节 | 继续润色 |
| 规划中断 | 有未完成规划 | 完成规划 |

```

---

## 状态检查

### 检查清单

```markdown
□ MEMORY.md 存在
□ 章节正文/ 目录存在
□ 大纲.md 存在（如果是新书）
□ 人物设定/ 目录存在（如有必要）

IF 任何检查失败：
→ 输出警告，询问用户是否创建/恢复
```

---

## 错误处理

### MEMORY.md 损坏

```markdown
⚠️ 检测到 MEMORY.md 损坏

可能的解决方案：
1. 从章节正文恢复（扫描章节文件）
2. 从 Git 历史恢复（如果有版本控制）
3. 重新开始（谨慎选择）

请选择 [1/2/3]：
```

### 章节文件丢失

```markdown
⚠️ 第{XXX}章文件未找到

当前状态：
- 章节计划：共{数字}章
- 已完成：{数字}章

建议：
- 如果已写完但文件丢失，请手动恢复文件
- 如果未写完，请继续从第{XXX}章开始

继续写第{XXX}章？ [Y/n]
```

## 禁止事项

- ❌ 不检查 MEMORY.md 直接恢复
- ❌ 虚构上下文
- ❌ 跳过状态检查直接写
- ❌ 忽略中断类型检测
