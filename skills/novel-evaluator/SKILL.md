# Novel Evaluator — 小说审稿评分系统

> 7 维量化评分引擎。从 ainovel-cli 七维评审借鉴而来（设定一致性/角色行为/节奏/叙事连贯/伏笔/钩子/审美品质），融合 traps-archive 47 条陷阱检测。每项发现必须逐条引用原文行号。

## 激活条件

- 用户要求审稿（`/evaluate` 命令）
- Writer 提交章节后进入审稿阶段
- 需要量化评分判断是否通过

## 7 维评分体系

| 维度 | 权重 | 检查要点 |
|------|------|---------|
| **情节** | 22% | 逻辑自洽、推进节奏、冲突设计、伏笔埋设/回收、高潮有力 |
| **人物** | 18% | 性格一致性、成长弧线、对话符合身份、配角独立行为线 |
| **文笔** | 18% | 语言流畅度、句式变化、描写质感、AI 套路检测（对照 47 条陷阱 §1-15） |
| **世界观** | 12% | 设定一致性、能力体系自洽、时间地理逻辑、社会结构合理 |
| **钩子** | 10% | 章节结尾悬念强度、"追读欲"、钩子合理性（非强行断章）、多线钩子交织 |
| **情感** | 10% | 情绪递进自然、共情力、人物关系可信度、情感层次 |
| **创新** | 10% | 设定新颖度、情节不可预测性、人物独特性、表达创新 |

**评分标准（每维 1-10 分）：**
- 9-10: 优秀 — 有明显亮点，超出预期
- 7-8: 良好 — 达标，无明显问题
- 5-6: 一般 — 有可改进之处
- 3-4: 较差 — 有明显缺陷
- 1-2: 很差 — 严重影响阅读体验

**加权公式：**
```
总分 = 情节×0.22 + 人物×0.18 + 文笔×0.18 + 世界观×0.12 + 钩子×0.10 + 情感×0.10 + 创新×0.10
```

> **为什么"钩子"权重独立？** 网文连载模式下，章节结尾的悬念是留存读者的核心。ainovel-cli 将"钩子"作为 Editor 七维评审中的一个独立维度，因为它直接影响次日追读率。传统文学审稿中钩子常归入"情节"维度，但网文的连载属性要求将钩子单独考量——某章情节总体不错但结尾平淡，在传统评审中可能通过，在连载评审中应被标记。

## 通过阈值

| 分数 | 结论 | 处理 |
|------|------|------|
| ≥80 | **APPROVE** | 优秀，直接进入润色 |
| 70-79 | **APPROVE** | 通过，可进入润色（附带 minor suggestions） |
| 60-69 | **BLOCK** | 不通过，返修（标注 Important issues） |
| <60 | **BLOCK** | 严重不通过，返修（标注 Critical issues） |

单章最多返修 2 次。2 次返修后仍未通过 → 输出最终审查报告，提示用户介入。

> **钩子专项门禁**：钩子维度 <5 分时，即使总分 ≥70，仍需标注为 Important issue 并要求强化章节结尾。

## 问题定级

| 级别 | 含义 | 示例 |
|------|------|------|
| **Critical** | 功能性问题 | 人物性格崩塌、时间线矛盾、伏笔冲突、世界观设定违背 |
| **Important** | 影响阅读体验 | AI 套路化表达过多、节奏失衡、对话不符合人物、钩子弱 |
| **Suggestion** | 可改进 | 句式可更多变、环境描写可更具体 |
| **Nit** | 细节 | 个别词语可替换、分段可优化 |

## 举证规则（借鉴 ainovel-cli 逐条引用原文）

**所有 Critical 和 Important 级别的发现，必须同时提供以下三项，缺少任一则该发现降级或丢弃：**

1. **原文引用**：引用具体段落/行号（如 "第 45 行：'眼中闪过一丝决然'"）
2. **失败场景**：描述触发什么条件会导致问题（如 "AI 高频表情 #8，单章出现 3 次，破坏人类作者语感"）
3. **现有防护为何不足**：说明为什么现有的人物设定/大纲/框架约束未能防止此问题

**Vague findings 处置规则：**
- 无法引用原文行号的 → **丢弃**（不是 actionable 的审查）
- 只写"某处有问题"不写行号的 → **降级**（Critical → Important, Important → Suggestion）
- "建议考虑…"类无触发场景的空泛建议 → **丢弃**

## 与 traps-archive 联动

审稿时自动对照 `traps-archive/novel/00-all.md` 的 47 条陷阱：

- **§ AI痕迹/套路化表达** (1-15): 检查"眼中闪过一丝"、"嘴角勾起一抹"、"深吸一口气"等高频 AI 表达
- **§ 人设崩塌/声音趋同** (16-21): 检查角色行为是否符合设定、对话是否雷同
- **§ 节奏失控** (22-27): 检查开场信息轰炸、平淡章节、高潮后急转直下
- **§ 逻辑漏洞/伏笔不回收** (28-33): 检查时间线矛盾、伏笔遗忘、世界观矛盾
- **§ 章节结构/悬念设计** (34-40): 检查章节末尾钩子、篇幅、分段、开头重复
- **§ 题材特定** (41-47): 根据题材检查对应陷阱

每条陷阱命中即记录到 Findings，标注严重级别和**原文行号**。

## 工作流程

1. **读取原文** → 读待审查章节全文
2. **7 维评分** → 每维打分并**附具体行号引用**（如 "情节 7 分 — 第 12-45 行推进有力，但 67-72 行节奏拖沓"）
3. **陷阱扫描** → 对照 47 条陷阱逐条检查，命中项附行号
4. **问题定级** → 对发现的问题定级，Critical/Important 须满足举证规则三要素
5. **修改建议** → 每条问题附具体修改方向 + 正面示例
6. **结论判定** → APPROVE / BLOCK

```markdown
| 创新 | 6 | 10% | 0.60 | 开篇方式较常规 |

- [Important] #8 "眼中闪过一丝" 出现 3 次 — 行 45, 112, 278

  - 现有防护不足：writer 自检未覆盖，MEMORY.md 无禁用词登记
- [Important] #35 章节末尾没有钩子 — 行 412-418
  - 失败场景：以"今天就这样了"收尾 → 读者无追读欲望
  - 现有防护不足：大纲未强制要求钩子设计

### 修改建议
1. [Important] 行 45/112/278：替换"眼中闪过一丝"为具体肢体语言，如：
   - 行 45 "眼中闪过一丝决然" → "他攥紧剑柄，指节泛白"
   - 行 112 "眼中闪过一丝慌乱" → "脚步后退半步，撞翻了桌上的茶盏"
2. [Important] 行 412-418：强化章节结尾钩子，建议用"信息差悬念"（主角已知但读者不知的危险）或"反转型钩子"（新信息颠覆当前认知）

### 结论: BLOCK (60-69 档，2 个 Important issues 须修复)
```

## 零发现审查

**干净的审查是有效审查。** 如果本章在 7 维全部达标且无陷阱命中，必须输出：

```markdown
### 评分
| 维度 | 分数 |
|------|------|
| ... 7 维均 ≥7 ... |
| **总分** | **7.XX** |

### 陷阱命中
无

### 结论: APPROVE — 零发现通过
```

不要为"显得有成果"而捏造 findings。

## 禁止

- ❌ 跳过评分直接给结论
- ❌ Critical/Important 发现无原文引用（行号/段落号）— 必须降级或丢弃
- ❌ 空泛建议（"建议优化"、"考虑改进"类无具体方案）
- ❌ 直接修改正文
- ❌ 未读 traps-archive 就审稿
- ❌ 捏造 findings 让审查看起来"有价值"
