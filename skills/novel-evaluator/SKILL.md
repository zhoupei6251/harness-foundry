# 6维量化评分

## 激活条件

- 用户要求审稿（"审稿第X章"、"评分"、"检查质量"）
- 写完章节后进入审稿环节
- 需要质量评估报告

## 核心能力

### 6维评分体系

| 维度 | 评分标准 |
|------|---------|
| 情节 | 情节推进是否合理、有吸引力 |
| 人物 | 人物塑造是否立体、对话是否符合身份 |
| 文笔 | 文笔是否流畅、是否有 AI 味 |
| 世界观 | 世界观设定是否一致、细节是否合理 |
| 情感 | 情感表达是否真挚、有层次 |
| 创新 | 是否有创新点、避免套路化 |

### 问题定级

- **严重**：必须修改，影响整体质量
- **一般**：建议修改，提升质量
- **轻微**：可选修改，锦上添花

### 通过判断

- 6维平均分 ≥7 分 → 通过
- 任一维度 <5 分 → 不通过
- 严重问题 ≥3 个 → 不通过

## 工作流程

1. **读取稿件**
   - 读取待审章节
   - 读取 `MEMORY.md`：人物状态、伏笔
   - 读取 `大纲.md`：本章目标

2. **6维评分**
   - 逐维度评分（1-10 分）
   - 记录评分依据
   - 标记问题段落

3. **问题定级**
   - 列出问题清单
   - 按严重程度定级
   - 提出修改建议

4. **通过判断**
   - 计算平均分
   - 判断是否通过
   - 输出审查报告

## 禁止事项

- ❌ 直接改正文
- ❌ 跳过评分直接给结论
- ❌ 无依据的模糊评价
- ❌ 审完后自己改稿

## 产物

- 审查报告（评分+问题清单+修改建议+通过/不通过）
- 返修时交回 writer
