# Novel Voice Profile — 人物声音档案

> 借鉴 ECC brand-voice 的 Source-First 方法论。
> 从角色已有的出场内容中抽取语言特征，建立可复用的人物声音档案。
> **每个重要角色都应该有独立的 Voice Profile。**

## 核心概念

```
ECC brand-voice 的方法 → novel 对应

1. Source-First  ─→ 从已有章节收集角色对话样本
2. Voice Profile ─→ 建立角色的语言特征档案
3. Hard Bans     ─→ 该角色绝对不能说的话
4. Downstream Use ─→ 写该角色时加载 Profile
```

---

## 何时使用

| 场景 | 触发条件 |
|------|----------|
| 创建新角色 | 写人物设定时同步创建 Voice Profile |
| 写对话前 | 加载该角色的 Voice Profile |
| 审稿发现"声音趋同" | 检查 Profile 是否缺失或不准确 |
| 新角色首次说话 | 根据性格设定推导 Voice Profile |

## 采集方法（借鉴 ECC Source-First）

### Step 1: 收集样本

```markdown
从已写章节收集该角色所有对话：

1. 导出角色所有台词
2. 按出场顺序排列
3. 标注对话场景（日常/战斗/内心独白）

最少需要：5-10 段对话样本
```

### Step 2: 提取特征

```markdown
分析样本：

1. 句长统计
   - 最长句子：{N}字
   - 最短句子：{M}字
   - 平均句长：{AVG}字

2. 高频词
   - Top 5 用词
   - 独有词汇（其他角色不用的）

3. 句式模式
   - 反问句占比：{X}%
   - 感叹句占比：{Y}%
   - 陈述句占比：{Z}%
```

### Step 3: 建立 Profile

```markdown
根据分析结果填写 Voice Profile 模板。

关键原则（借鉴 ECC）：
- "Specificity beats adjectives" — 具体特征比形容词描述更有用
- "One profile, reusable" — 一份 Profile 整本书复用
- "The point is operational reuse" — 目的是可操作，不是文学评论
```

### Step 4: 验证与迭代

```markdown
写完新章节后验证：

1. 检查角色对话是否与 Profile 一致
2. 如果角色成长导致说话方式变化，更新 Profile
3. 每 5 章重新审查一次 Profile
```

## 使用方式

### 写作前

```markdown
加载该角色 Voice Profile：
- 在写对话时对照 Profile 的 Hard Bans
- 确认对话行为（手势、眼神、神态）
- 确认人物关系（对谁说什么话）
```

### 写作中

```markdown
快速检查：
- 这句话符合该角色的句长习惯吗？
- 用了被禁用的词汇吗？
- 和其他角色的对话有区分度吗？
```

### 写作后

```markdown
Voice Profile 一致性审查（借鉴 ECC Quality Gate）：
□ 该角色所有对话符合 Profile
□ Hard Bans 无触发
□ 情绪变化后的说话方式变化符合 Profile
□ 与其他角色（特别是同一场景的）有明显语音区分
```

## 常见问题

### 问题 1：角色出场太少，样本不够

```markdown
类似角色法：
- 找到该角色的原型/参考角色
- 从参考角色的对话中提取特征
- 标注为"临时 Profile — 待完善"

原则：宁可有粗略的 Profile，也不能没有 Profile
```

### 问题 2：角色成长后说话方式变了

```markdown
版本管理：
- 保留旧 Profile（标注"第X章前"）
- 创建新 Profile（标注"第X章后"）
- 在 Profile 中说明变化原因

目的：变化应该有依据（经历了某事件 → 性格变化 → 说话方式变化）
```

### 问题 3：两个角色声音太像

```markdown
差异化检查：
- 如果删除角色名只看对话，能分出来是谁在说话吗？
- 如果不能 → 调整语言特征

具体方法：
- 改变句长习惯（一个说短句，一个说长句）
- 改变用词风格（一个口语化，一个书面化）
- 给其中一个角色独特的口头禅或表达习惯
```

## 批量生成

### 从人物设定批量生成

```markdown
# 输入：人物设定文件
# 输出：所有主要角色的 Voice Profile

## 生成流程
1. 读取人物设定（性格、身份、背景）
2. 基于性格推导语言特征
3. 标注"未基于实际样本，待完善"
4. 首次出场后根据实际对话更新
```

---

## 禁止事项

- ❌ 不建立 Profile 就开始写角色对话
- ❌ 用同一个 Profile 给所有角色（"所有人说话都一样"）
- ❌ Profile 中有矛盾描述（"说话快"同时"每句停顿"）
- ❌ Profile 中 Hard Bans 为空（每个角色都应该有避免说的话）
- ❌ 不实际使用（建立了 Profile 但写对话时不看）
- ❌ Profile 在角色成长后不更新
