# 自我进化命令

> 分析会话中的错误、纠正和最佳实践，自动提取新的陷阱和模式

## 触发

```
/evolve
```

## 执行流程

### 1. 扫描会话历史
- 读取最近 N 次会话的 MEMORY.md
- 提取用户纠正、错误修复、最佳实践

### 2. 分类分析

#### 陷阱候选（Trap Candidates）
识别以下模式：
- 用户纠正了 AI 的错误
- 重复出现的 bug 或问题
- 违反规则的行为
- 性能问题或反模式

**提取条件**：
- 同一问题出现 ≥ 2 次
- 用户明确说"不要..."或"应该..."
- 导致返工或修复

#### 模式候选（Pattern Candidates）
识别以下模式：
- 用户表扬的做法
- 高效的解决方案
- 可复用的代码结构
- 最佳实践

**提取条件**：
- 用户说"很好"、"就是这样"
- 一次通过的实现
- 可复用的抽象

#### 规则候选（Rule Candidates）
识别以下模式：
- 反复需要提醒的约束
- 隐含的项目规范
- 未文档化的约定

**提取条件**：
- 每次都需要用户提醒
- 多个会话重复说明
- 违反后需要修复

### 3. 生成建议

输出格式：

```markdown
## 进化分析报告

### 陷阱候选（3 个）

1. **禁止在循环中查询数据库**
   - 出现次数：3
   - 相关会话：2024-01-15, 2024-01-18, 2024-01-20
   - 建议：添加到 `rules/code/common/performance.md`

2. **API 响应必须包含错误码**
   - 出现次数：2
   - 相关会话：2024-01-16, 2024-01-19
   - 建议：添加到 `rules/code/common/api.md`

### 模式候选（2 个）

1. **使用 Result 类型处理错误**
   - 出现次数：4
   - 相关会话：2024-01-14, 2024-01-17, 2024-01-19, 2024-01-21
   - 建议：添加到 `references/patterns.md`

2. **组件状态管理使用 useReducer**
   - 出现次数：2
   - 相关会话：2024-01-17, 2024-01-20
   - 建议：添加到 `references/patterns.md`

### 规则候选（1 个）

1. **所有 API 必须有版本前缀**
   - 出现次数：3
   - 相关会话：2024-01-15, 2024-01-18, 2024-01-21
   - 建议：添加到 `rules/code/common/api.md`
```

### 4. 用户确认

展示分析报告，等待用户确认：

```
是否应用以上建议？
- [y] 全部应用
- [n] 全部跳过
- [s] 选择性应用（逐个确认）
```

### 5. 应用变更

根据用户选择：

#### 应用陷阱
- 添加到 `traps-archive/code/00-all.md` 或对应域
- 更新 `references/traps.md` 致命版（如果达到 25 条阈值）

#### 应用模式
- 添加到 `references/patterns.md`
- 如果是技术栈特定，添加到 `rules/code/<tech>/patterns.md`

#### 应用规则
- 添加到对应的 `rules/code/<tech>/` 文件
- 如果是通用规则，添加到 `rules/code/common/`

### 6. 提交变更

```bash
git add traps-archive/ references/ rules/
git commit -m "evolve: 自动提取会话经验 (陷阱: 3, 模式: 2, 规则: 1)"
```

## 参数

```
/evolve                    # 分析最近 10 次会话
/evolve --sessions 20      # 分析最近 20 次会话
/evolve --domain code      # 只分析代码域
/evolve --dry-run          # 只生成报告，不应用变更
```

## 注意事项

- 只分析 MEMORY.md 中记录的内容
- 不会修改用户已确认的规则
- 所有变更需要用户确认
- 建议定期运行（每周一次）
