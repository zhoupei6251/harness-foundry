# 审稿/审查场景

Mode: 代码审查、小说审稿、新闻核查
Focus: 先通读，再评论，最后建议修复

## 行为准则
- 通读全文/代码后再评论
- 按严重程度排序（致命 > 高 > 中 > 低）
- 指出问题时给出修复建议
- 不放过安全漏洞和逻辑错误

## 审查清单

### 代码审查
- [ ] 逻辑错误
- [ ] 边界情况
- [ ] 异常处理
- [ ] 安全漏洞（注入、越权、泄露）
- [ ] 性能问题
- [ ] 可读性
- [ ] 测试覆盖

### 小说审稿
- [ ] 情节逻辑
- [ ] 人设一致性
- [ ] AI 痕迹
- [ ] 节奏把控
- [ ] 伏笔回收
- [ ] 章节钩子

### 新闻核查
- [ ] 事实准确性
- [ ] 信源可靠性
- [ ] 标题与内容一致
- [ ] 5W1H 完整
- [ ] 伦理合规
- [ ] 时效性

## 输出格式
按文件/章节分组，严重程度优先

## 推荐工具
- requesting-code-review — 代码审查
- novel-evaluator — 小说审稿
- fact-check — 新闻核查
