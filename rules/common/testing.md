# 通用测试规则

## 测试原则
- 测试独立，不依赖执行顺序
- 测试可重复执行
- 测试覆盖正常路径和异常路径
- 测试名称清晰描述场景

## 测试结构
- Given-When-Assert 模式
- 每个测试只测一个行为
- 避免测试间共享状态
- 使用 fixture 管理测试数据

## 覆盖率
- 核心业务逻辑 ≥ 80%
- 工具函数 ≥ 90%
- 关键路径 100%
- 避免为覆盖率而写无意义测试

## Mock 使用
- Mock 外部依赖（API、数据库、文件系统）
- 避免过度 Mock
- Mock 行为需验证
- 集成测试减少 Mock

## 测试类型
- 单元测试：测试单个函数/方法
- 集成测试：测试模块间交互
- E2E 测试：测试完整用户流程
- 性能测试：测试响应时间和吞吐量
