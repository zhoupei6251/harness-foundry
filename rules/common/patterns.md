# 通用编码规范

## 命名规范
- 变量/函数：camelCase（JS/TS）或 snake_case（Python/Go）
- 类/接口：PascalCase
- 常量：UPPER_SNAKE_CASE
- 文件名：kebab-case
- 避免缩写，使用完整单词

## 代码组织
- 单一职责原则：一个函数做一件事
- 函数长度不超过 50 行
- 文件长度不超过 500 行
- 避免深层嵌套（最多 3 层）

## 错误处理
- 显式处理错误，不忽略
- 错误信息包含上下文
- 使用自定义错误类型
- 日志记录完整堆栈

## 注释规范
- 注释解释"为什么"而非"做什么"
- 公共 API 必须有文档注释
- 避免过时注释
- 使用 TODO/FIXME 标记待办

## Git 规范
- 提交信息使用约定式格式（feat/fix/docs）
- 提交粒度小，逻辑完整
- 避免提交敏感信息
- 分支命名规范（feature/bugfix/hotfix）
