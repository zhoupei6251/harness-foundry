# 通用 Hooks 规则

## Hook 设计原则
- 单一职责：一个 Hook 做一件事
- 快速执行：避免阻塞主流程
- 明确输出：成功/失败/警告清晰区分
- 可配置：支持启用/禁用

## Hook 类型

### PreToolUse（工具执行前）
- 用途：验证、提醒、阻止
- 退出码：0（继续）/ 2（阻止）
- 示例：编译检查、格式化、安全扫描

### PostToolUse（工具执行后）
- 用途：分析、通知、记录
- 退出码：0（成功）
- 示例：测试运行、质量检查、日志记录

### Stop（响应结束后）
- 用途：批量处理、状态保存
- 退出码：0（成功）
- 示例：格式化、类型检查、记忆同步

## Hook 最佳实践
- 使用异步 Hook 处理耗时操作
- 设置合理的超时时间
- 错误信息要 actionable
- 避免 Hook 间相互依赖

## Hook 配置
- 按环境配置（dev/test/prod）
- 支持全局和项目级配置
- 提供默认值和覆盖机制
- 文档化所有可用 Hook
