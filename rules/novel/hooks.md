# Novel Hooks — 创作钩子规则

> 定义 novel 域的 PreToolUse、PostToolUse、Stop 行为规则。
> 与 `hooks/hooks.json` 中的 hook 配置一一对应。

## PreToolUse Hooks

### H1: 设定一致性检查

| 项 | 内容 |
|---|---|
| **触发** | 写新章节前（Write 到 `章节正文/*.md`） |
| **行为** | 读取 MEMORY.md、大纲.md、人物设定/ |
| **检查** | 新章节不违背已有设定、承接上章结尾 |
| **失败处理** | 暂停写作，向用户报告冲突 |

### H2: 章节目标检查

| 项 | 内容 |
|---|---|
| **触发** | 写新章节前 |
| **行为** | 检查大纲中本章目标、伏笔安排 |
| **检查** | 有明确情节推进？有伏笔埋设/回收？ |
| **失败处理** | 向用户确认本章目标 |

---

## PostToolUse Hooks

### H3: 字数检查（硬性要求）

| 项 | 内容 |
|---|---|
| **触发** | 写完章节后（Write 到 `章节正文/*.md`） |
| **行为** | Python 脚本统计字数（包含标点） |
| **阈值** | ≥2000 字 |
| **失败处理** | 警告并要求扩写至达标 |

### H4: AI 痕迹检测

| 项 | 内容 |
|---|---|
| **触发** | 写完章节后 |
| **行为** | 扫描高频 AI 词汇 |
| **检测列表** | `眼中闪过`、`嘴角勾起`、`深吸一口气`、首先/其次/最后、仿佛/似乎、不是A不是B而是C、车轱辘话 |
| **失败处理** | 标记段落，要求润色时消除 |

### H5: 钩子检查

| 项 | 内容 |
|---|---|
| **触发** | 写完章节后 |
| **行为** | 检查章节结尾 |
| **合格标准** | 有悬念/转折/冲突未解决 |
| **不合格标志** | "今天就这样了"、"本章完"、"预知后事如何" |
| **失败处理** | 警告，要求强化结尾钩子 |

### H6: 人物声音检查

| 项 | 内容 |
|---|---|
| **触发** | 写完章节后 |
| **行为** | 检查各角色对话风格 |
| **检查** | 不同角色说话风格是否差异化 |
| **失败处理** | 标记，要求润色时调整 |

---

## Stop Hooks

### H7: 记忆同步

| 项 | 内容 |
|---|---|
| **触发** | 会话结束时 |
| **行为** | 更新 MEMORY.md |
| **必填项** | 章节摘要、人物状态变更、伏笔状态、下章待推进 |
| **文件** | `章节正文/MEMORY.md` |

### H8: 标点规范检查

| 项 | 内容 |
|---|---|
| **触发** | 会话结束时 |
| **行为** | 检查章节正文标点 |
| **合格标准** | 全角标点（，。？！……） |
| **不合格标志** | 半角逗号、句号、引号、感叹号 |
| **失败处理** | 要求修改 |

---

## 禁止事项

- ❌ PreToolUse 跳过设定读取直接写
- ❌ PostToolUse 跳过字数检查直接交付
- ❌ AI 痕迹检测发现高频词不标记
- ❌ 章节结尾无钩子不警告
- ❌ 会话结束不更新 MEMORY.md
