# 常见陷阱（致命版）

> 每个域 25 条最致命陷阱。完整版见 `traps-archive/<domain>/00-all.md`

---

## 代码域（25 条）

| # | 陷阱 | 避免方式 |
|---|------|----------|
| 1 | AOP 自调用（`this.xxx()`）不走代理，事务失效 | 注入自己 `self.xxx()` |
| 2 | `@Async` + `@Transactional` 组合失效 | 异步用 `TransactionTemplate` |
| 3 | 事务内缓存更新，事务回滚缓存已变 | 用 `afterCommit` 回调 |
| 4 | 锁还没提交就释放，读到旧数据 | `isHeldByCurrentThread()` + defer 释放 |
| 5 | 多实例用本地锁形同虚设 | 用 Redisson 分布式锁 |
| 6 | `@Async` / MQ / Feign 调用丢失租户上下文 | 用 `TaskDecorator` / header 传递 |
| 7 | 水平越权——改 id 查他人数据 | 所有查询带租户 + 归属校验 |
| 8 | SQL 注入 `${xxx}` | 用 `#{}` |
| 9 | 后端相信前端价格/金额 | 后端计算 |
| 10 | 接口无频率限制被刷 | Redis 限流 |
| 11 | 循环依赖启动失败 | 用 `@Lazy`；禁止 common → 业务模块 |
| 12 | `@ConfigurationProperties` 绑定失败静默用默认值 | 加 `@Validated` |
| 13 | 虚拟线程内 `synchronized` 阻塞平台线程 | 用 `ReentrantLock` |
| 14 | 深分页 LIMIT offset | 游标分页 |
| 15 | N+1 循环 SQL | 批量查询 |
| 16 | 乐观锁冲突频繁 | 高并发用悲观锁 |
| 17 | 热 Key 打单节点 | Redis Cluster + 本地缓存 |
| 18 | 分布式锁非原子（setnx+expire） | 用 Redisson |
| 19 | 锁 TTL 过短自动释放 | 看门狗续期；TTL = 预估 × 2~3 |
| 20 | 缓存 null 值空查询穿透 | 特殊标记或短 TTL + 锁 |
| 21 | 过度设计——一次调用建工厂/策略 | 直接写；第二次才抽象 |
| 22 | 空 catch 块吞异常 | 处理或转译上抛 |
| 23 | Controller 写业务逻辑 | 只做参数校验 + 路由 |
| 24 | TraceId 异步线程丢失 | MDC.put 贯穿全链路 |
| 25 | 敏感信息明文日志 | 脱敏或 `[REDACTED]` |

## 小说域（25 条）

| # | 陷阱 | 避免方式 |
|---|------|----------|
| 1 | 滥用"首先/其次/最后"机械罗列 | 自然叙事过渡，不用列举式连接词 |
| 2 | 堆砌华丽辞藻、无实质内容 | 用具体细节替代抽象形容词 |
| 3 | 车轱辘话——同一意思反复表达 | 删除冗余，每句话承载新信息 |
| 4 | "就在这时""突然之间""只见"等俗套过渡 | 用动作/环境变化自然过渡 |
| 5 | 万能结尾——每章"预知后事如何" | 用具体悬念钩子替代套话 |
| 6 | 人物情感表达单一 | 通过行为、微表情、内心独白多维展现 |
| 7 | 对话过于直白或书面化，不符合人物身份 | 根据角色性格调整对话风格 |
| 8 | "眼中闪过一丝"——AI 高频表情描写 | 用具体的肢体语言替代 |
| 9 | "嘴角勾起一抹"——AI 高频微笑描写 | 用更自然的表情变化 |
| 10 | "深吸一口气"——AI 高频情绪前奏 | 直接写人物行动或心理 |
| 11 | 不同角色说话风格雷同 | 给每个角色定义独特语言习惯 |
| 12 | 角色行为不符合既定性格 | 写动作前查人物设定 |
| 13 | 主角突然降智推动剧情 | 用合理方式制造困境 |
| 14 | 反派脸谱化——为坏而坏 | 给反派合理的动机和逻辑 |
| 15 | 配角工具人——只在需要时出现 | 给配角独立的行为线 |
| 16 | 开场信息轰炸——第一章塞入过多设定 | 设定分散到剧情中逐步揭示 |
| 17 | 平淡章节过长——连续几章无冲突 | 每章至少一个小冲突或转折 |
| 18 | 机械降神——困境用突然能力解决 | 提前埋下伏笔或能力来源 |
| 19 | 伏笔埋了但后面忘了回收 | 伏笔登记表，标注回收章节 |
| 20 | 时间线矛盾——前后时间对不上 | 维护时间线文档 |
| 21 | 世界观前后不一致——设定自相矛盾 | 维护设定文档，写前查阅 |
| 22 | 能力体系崩坏——战力忽高忽低 | 明确能力等级和限制条件 |
| 23 | 章节末尾没有钩子 | 每章结尾留悬念或转折 |
| 24 | 章节过长（>8000字）无分段感 | 大章分小节，用分隔线或空行 |
| 25 | 全章都是对话或全是描写，缺乏变化 | 对话、动作、心理交错使用 |

## 新闻域（25 条）

| # | 陷阱 | 避免方式 |
|---|------|----------|
| 1 | 单一信源就发布 | 至少两个独立信源确认 |
| 2 | 引用社交媒体内容未核实 | 联系当事人或官方渠道确认 |
| 3 | 过时信息当新闻 | 核实事件发生日期 |
| 4 | 断章取义——引用时脱离上下文 | 引用完整语境 |
| 5 | 数据/统计数字来源不明 | 标明数据来源链接或出处 |
| 6 | 把传言当事实报道 | 明确标注"据传""未经证实" |
| 7 | 翻译错误——外文报道机翻导致曲解 | 人工核对原文含义 |
| 8 | 专家身份未核实——随意冠名"专家说" | 确认受访者的专业背景 |
| 9 | 标题党——标题与正文不符 | 标题准确反映正文核心 |
| 10 | 夸大影响——"震惊""史上最大"滥用 | 用准确数据和描述 |
| 11 | 把可能性当必然性 | 保留不确定性措辞 |
| 12 | 暗示未证实的因果关系 | 区分相关性和因果性 |
| 13 | 标题带情绪引导——先入为主 | 标题保持中立客观 |
| 14 | 报道中包含个人隐私信息 | 隐去姓名、地址、身份证号 |
| 15 | 偏见性报道——只呈现一方观点 | 平衡呈现多方立场 |
| 16 | 版权侵犯——直接搬运其他媒体内容 | 注明来源，合理引用 |
| 17 | 煽动性语言——挑起对立情绪 | 使用中性词汇，避免情绪化表达 |
| 18 | 灾难/事故报道中描写过于血腥 | 用克制的语言描述事实 |
| 19 | 导语过长——超过100字还没到重点 | 导语1-2句话，点明核心 |
| 20 | 倒金字塔结构被打破——重要信息放最后 | 最重要信息在前，细节在后 |
| 21 | 缺少5W1H | 发稿前检查 who/what/when/where/why/how |
| 22 | 专业术语不解释 | 首次出现时加简短说明 |
| 23 | 段落过长——一个段落超过5行 | 每段一个核心信息 |
| 24 | 追热点但角度同质化 | 找独特切入角度 |
| 25 | 热点信息混乱时抢发——出错率高 | 先核实再发稿，宁晚勿错 |
