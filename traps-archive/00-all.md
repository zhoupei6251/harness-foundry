## Spring Boot / RuoYi 特有陷阱

| # | 类别 | 陷阱描述 | 如何避免 | 发现 |
|---|------|---------|---------|------|
| 15 | AOP | AOP 自调用（`this.xxx()`）不经过代理，事务/@Cache/@Async 全部失效 | 用 `((Service) AopContext.currentProxy()).xxx()` 或注入自己 `self.xxx()` | 2026-06-18 |
| 16 | 事务 | `@Async` 方法内的 `@Transactional` 失效——异步线程不在事务上下文里 | `@Async` 方法内不要依赖事务传播；需要事务的异步操作用 `TransactionTemplate` 编程式事务 | 2026-06-18 |
| 17 | 事务 | `REQUIRED` 传播下，内部方法调自己不走代理——多个 Service 互调时事务丢失 | 同一类内部调用不走 Spring 代理。跨类调用或用注入自己的代理 | 2026-06-18 |
| 18 | 配置 | `@ConfigurationProperties` 绑定失败静默用默认值——配置没生效但程序不报错 | 启动时加 `@Validated`，加 `@EnableConfigurationProperties` 显式启用；启动后检查绑定值 | 2026-06-18 |
| 19 | 循环依赖 | Service A 注入 B，B 注入 A → 启动失败或运行时随机 BeanCurrentlyInCreationException | 用 `@Lazy` 注入打破循环；或重构为单向依赖；禁止 common 模块依赖业务模块 | 2026-06-18 |
| 20 | 循环依赖 | 模块间循环 import（common → aigc → common）→ 编译时好运行时炸 | common 只写纯工具和接口；业务模块不依赖 common 业务类 | 2026-06-18 |
| 21 | 缓存 | `@Cacheable` 的 key 用对象引用——同一个对象的不同实例 hashCode 不同，缓存永远 miss | key 用稳定值（id、name），不要用整个对象 | 2026-06-18 |
| 22 | 缓存 | Redis 缓存了 null 值导致空查询穿透——大量请求同时查不存在的数据 | 缓存空值用特殊标记（如 `NULL_VALUE`）；或短 TTL + 分布式锁 | 2026-06-18 |
| 23 | MyBatis | `WHERE id IN (...)` 传空集合导致 SQL 语法错误 | 用 `if (ids.isEmpty()) return;` 提前拦截；或 `AND 1=0` 作为 fallback | 2026-06-18 |
| 24 | MyBatis | `WHERE deleted = 0` 忘记加——查出了软删的数据 | 用 MyBatis-Plus 的 `isDeleted()` 或 `@Where` 注解；Mapper XML 中手动加 `del_flag = 0` | 2026-06-18 |
| 25 | 线程池 | 自定义 `@Async` 不指定线程池——用默认的 SimpleAsyncTaskExecutor（无上限）→ OOM | 自定义线程池：`ThreadPoolTaskExecutor` + `@EnableAsync(pool = "...")` 或全局配置 | 2026-06-18 |
| 26 | 线程池 | 虚拟线程内阻塞操作——JDK21 虚拟线程绑定了平台线程，阻塞会导致虚拟线程泄漏 | 虚拟线程内禁止 `synchronized`、阻塞 IO；用 `ReentrantLock` 替代 `synchronized` | 2026-06-18 |
| 27 | 异常 | 全局异常处理器捕获了业务异常但返回 500——异常类型不匹配 | 检查 `GlobalExceptionHandler` 是否漏掉了特定异常类型；业务异常用统一错误码枚举 | 2026-06-18 |
| 28 | 异常 | 抛 `RuntimeException` 而不用业务异常——前端无法区分错误类型 | 用项目统一的 `BusinessException` 或错误码枚举；`GlobalExceptionHandler` 统一处理 | 2026-06-18 |
| 29 | 参数校验 | `@Valid` 校验失败但没配 GlobalExceptionHandler → 500 而非 400 | 确认 `GlobalExceptionHandler` 有 `MethodArgumentNotValidException` 处理；或 Spring Boot 2.3+ 需手动加 `spring-boot-starter-validation` | 2026-06-18 |
| 30 | 热加载 | DevTools 引起的类加载器问题——Bean 注入在热加载后失效 | Dev 期间避免用 `Bean.getClass()` 做类型判断；热加载后重启 Bean 上下文 | 2026-06-18 |

## 虚拟线程陷阱（JDK21）

| # | 类别 | 陷阱描述 | 如何避免 | 发现 |
|---|------|---------|---------|------|
| 31 | 虚拟线程 | 虚拟线程中 ThreadLocal 数据丢失——平台线程的 ThreadLocal 不自动复制到虚拟线程 | 用 `Thread.ofVirtual()` 创建虚拟线程前先传递上下文；或用框架封装的 Context 传递机制 | 2026-06-18 |
| 32 | 虚拟线程 | 虚拟线程中 InheritableThreadLocal 无法自动传递——子虚拟线程看不到父线程的租户上下文 | 用 RuoYi 的 `TenantContextHolder`（基于 `InheritableThreadLocal` 包装），确保在创建虚拟线程前调用 `set`；或用 Scoped Values | 2026-06-18 |
| 33 | 虚拟线程 | 虚拟线程跨线程传递时租户上下文丢失——IO 回调（如 CompletableFuture）在虚拟线程执行时拿不到租户 | 所有 IO 操作都在同一虚拟线程内完成；跨线程回调时用 `TenantContextHolder.set()` 重新设置 | 2026-06-18 |
| 34 | 虚拟线程 | 虚拟线程 + MyBatis 多数据源动态切换——数据源切换在线程结束时未清理，影响后续请求 | 切换数据源后确保用 `try-finally` 清理 `DataSourceContextHolder.clear()`；虚拟线程结束时必须清理 | 2026-06-18 |
| 35 | 虚拟线程 | 虚拟线程内 `synchronized` 锁住平台线程——一个虚拟线程阻塞会导致大量其他虚拟线程也被挂起 | 禁止在虚拟线程中使用 `synchronized`；用 `ReentrantLock` + `tryLock` 替代 | 2026-06-18 |
| 36 | 虚拟线程 | 虚拟线程 + `Thread.sleep()` 阻塞底层平台线程——看似轻量但实际绑定了平台线程 | 用 `LockSupport.parkNanos()` 或 `CountDownLatch.await()` 替代 `Thread.sleep()` | 2026-06-18 |
| 37 | 虚拟线程 | 虚拟线程 + NIO Channel 混用——虚拟线程的调度和 NIO 的 epoll 冲突 | 虚拟线程场景下优先用传统阻塞 IO（网络、文件），不要混用 NIO selector | 2026-06-18 |

## 性能陷阱

| # | 类别 | 陷阱描述 | 如何避免 | 发现 |
|---|------|---------|---------|------|
| 46 | 性能 | 深分页（offset 过大）——MySQL 先扫过前 N 条再返回， LIMIT 1000000,10 极慢 | 用游标分页（上一页最后一条 id）或 `WHERE id > lastId LIMIT 10`；或 ES 分页 | 2026-06-18 |
| 47 | 性能 | 全表扫描未加索引——慢查询日志里出现 `type=ALL` | `EXPLAIN` 分析查询；高频查询字段加索引；联合索引注意字段顺序 | 2026-06-18 |
| 48 | 性能 | `SELECT *` 拉取不需要的字段——网络传输量大、内存占用高 | 明确列出需要的字段；DTO/VO 只包含展示字段 | 2026-06-18 |
| 49 | 性能 | 乐观锁冲突频繁——高并发更新同一行时大量重试 | 评估并发程度；高并发场景用悲观锁 `SELECT FOR UPDATE`；乐观锁加失败重试次数限制 | 2026-06-18 |
| 50 | 性能 | 大批量插入未优化——循环单条 INSERT，事务大、速度慢 | 用 `INSERT INTO ... VALUES (...), (...), (...)` 批量插入；或 MyBatis-Plus `saveBatch` | 2026-06-18 |
| 51 | 性能 | 关联查询 N+1——先查主表再循环查子表，1 次变 N+1 次 | 用 `JOIN` 或批量 `IN` 查询；MyBatis-Plus 的 `fetchLatest` 预加载 | 2026-06-18 |
| 52 | 性能 | 不分页的列表查询——接口无分页参数，前端传大 pageSize 或不分页直接崩 | 列表接口必须限制最大 pageSize（如最多 100）；无分页接口加告警 | 2026-06-18 |
| 53 | 性能 | 大对象存 Redis——图片/文件序列化后存 Redis，内存爆炸 | 大对象存 OSS/CDN，Redis 只存 URL；大 JSON 压缩后存储 | 2026-06-18 |
| 54 | 性能 | 热 Key 打在单节点——热点数据集中在一个 Redis 节点，QPS 上不去 | 用 Redis Cluster 或本地缓存（Caffeine/Cache）分流；热点数据加本地缓存 | 2026-06-18 |
| 55 | 性能 | 连接池配置不当——MySQL 连接池 max 太小 QPS 受限，或虚拟线程场景下连接池过大 | 虚拟线程场景下连接池可适度调大（CPU 核数 * 2 ~ 4 倍）；根据压测调优 | 2026-06-18 |
| 56 | 性能 | 递归查询无限制——树形结构递归太深，栈溢出或查询超时 | 递归加 `LIMIT` 和深度限制；用循环替代递归；MySQL 递归深度不超过 10 层 | 2026-06-18 |
| 57 | 性能 | 循环内发 HTTP 请求——批量操作时代码写成 `for` 循环调外部 API，串行等待极慢 | 批量接口或并发 `CompletableFuture.allOf()`；单线程内串行改为并发 | 2026-06-18 |
| 58 | 性能 | 预热缓存失效导致雪崩——大批 key 同时过期，瞬间全打到 DB | 缓存过期时间加随机偏移量（±5% TTL）；或用永不过期 + 主动刷新 | 2026-06-18 |
| 59 | 性能 | 不合理的锁粒度——全局锁锁住整个方法，并发退化为串行 | 缩小锁范围；用分段锁；读写分离场景用 `StampedLock` 或 `ReadWriteLock` | 2026-06-18 |
| 60 | 性能 | 字符串拼接写 SQL——`"SELECT * FROM " + table + " WHERE id = " + id`，SQL 注入风险且无法复用执行计划 | 用 MyBatis `#{}` 参数化查询；动态表名用白名单校验 | 2026-06-18 |

## 安全陷阱

| # | 类别 | 陷阱描述 | 如何避免 | 发现 |
|---|------|---------|---------|------|
| 61 | 安全 | SQL 注入——`${xxx}` 直接拼接用户输入到 SQL，绕过了 MyBatis 的参数化保护 | 所有用户输入必须用 `#{}`；`${}` 仅限表名列名等可信场景并白名单校验 | 2026-06-18 |
| 62 | 安全 | 文件上传未校验类型——用户上传 JSP/ASP 等可执行文件，服务器沦陷 | 上传后校验文件头（Magic Number）；存储路径和 URL 不暴露真实路径；重命名文件 | 2026-06-18 |
| 63 | 安全 | 文件上传路径穿越——文件名含 `../` 导致写到 uploads 目录外 | 文件名必须 UUID/时间戳重命名；存储路径用绝对路径禁止相对路径拼接 | 2026-06-18 |
| 64 | 安全 | 水平越权——用户 A 通过修改 id 参数访问用户 B 的数据 | 所有查询必须带租户+用户归属校验；接口加归属校验注解或全局拦截器 | 2026-06-18 |
| 65 | 安全 | 垂直越权——普通用户通过改 role 参数提升权限 | 权限变更必须后端校验；前端传的角色不可信；加 `@RequiresPermissions` 注解 | 2026-06-18 |
| 66 | 安全 | 敏感数据明文日志——密码/Token/身份证/手机号打印到日志 | 日志输出前脱敏；敏感字段加 `@JsonIgnore` 或自定义序列化；日志审计脱敏 | 2026-06-18 |
| 67 | 安全 | 敏感数据返回前端——API 响应包含内部字段如密码盐值、内部 ID | VO/DTO 只包含前端需要的字段；密码相关字段永远不返回；用 `@JsonIgnore` | 2026-06-18 |
| 68 | 安全 | 接口无频率限制——注册/登录/发送验证码等接口可被无限调用 | 关键接口加 Redis 限流（IP + 用户维度）；图形验证码；操作冷却时间 | 2026-06-18 |
| 69 | 安全 | CORS 配置过宽——`Access-Control-Allow-Origin: *` 允许任意域请求 | 生产环境 CORS 白名单；credentials 为 true 时 origin 必须具体域名 | 2026-06-18 |
| 70 | 安全 | CSRF 跨站请求伪造——GET 请求有副作用，POST 未校验 CSRF Token | GET 请求禁止有副作用；状态变更必须 POST；Spring Security 启用 CSRF Token | 2026-06-18 |
| 71 | 安全 | SSRF 服务端请求伪造——接口接受用户传入的 URL 进行请求 | 用户传入的 URL 做域名白名单校验；禁止内网 IP（127.0.0.1、10.x.x.x、192.168.x.x） | 2026-06-18 |
| 72 | 安全 | 弱密码/默认密码——测试环境或内部系统用了弱密码或未改出厂默认密码 | 强制密码复杂度；首次登录强制改密；定期密码审计 | 2026-06-18 |
| 73 | 安全 | JWT 签名密钥硬编码——密钥写死在代码里，代码泄露即密钥泄露 | 密钥从配置中心/环境变量读取；定期轮换；不用相同的密钥签名不同服务 | 2026-06-18 |
| 74 | 安全 | 接口参数篡改——后端相信前端传来的价格/数量/金额等关键字段 | 关键业务字段必须后端计算/校验；订单金额从数据库读取，不从请求参数直接使用 | 2026-06-18 |

## Redis 陷阱

| # | 类别 | 陷阱描述 | 如何避免 | 发现 |
|---|------|---------|---------|------|
| 75 | Redis | Redis 连接未设置超时——网络抖动时线程阻塞在等待连接 | 连接池配置 `connectTimeout` + `soTimeout`（如 3000ms）；Jedis/Lettuce 都要配 | 2026-06-18 |
| 76 | Redis | Redis 大 Key——单个 Key 数据量超过 10MB，网络传输慢、内存倾斜 | 大数据拆分（如 Hash 结构过大）；或改用 DB 存储；BigKey 告警 + 异步删除 | 2026-06-18 |
| 77 | Redis | Redis 热 Key——单 Key QPS 过高打在单节点，Redis 变慢甚至宕机 | 用 Redis Cluster 分片；本地缓存 + Redis 热点探测；key 名字加随机后缀打散 | 2026-06-18 |
| 78 | Redis | Redis 持久化阻塞——RDB/AOF fork 时大数据量导致 stop-the-world | RDB 改为低峰期；AOF 用 `everysec` 而非 `always`；集群版关闭 RDB | 2026-06-18 |
| 79 | Redis | Redis 内存淘汰策略不当——内存达到 maxmemory 后写入失败而不是淘汰旧数据 | 根据业务选择淘汰策略（allkeys-lru/volatile-lru）；设置合理的 maxmemory 和淘汰策略 | 2026-06-18 |
| 80 | Redis | Redis Lua 脚本死循环——Lua 脚本中有无限循环或高耗时操作，阻塞所有命令 | Lua 脚本必须有终止条件；禁止 `while(true)`；大计算移出 Redis | 2026-06-18 |
| 81 | Redis | Redis pipeline 事务混用——`pipeline` 内又有 `MULTI/EXEC`，行为不符合预期 | 明确区分：批量命令用 pipeline；原子性用 Lua 或 `MULTI/EXEC`；不要嵌套 | 2026-06-18 |
| 82 | Redis | Redis 分布式锁误用——用 `setnx + expire` 非原子操作，加锁后 expire 前进程挂了导致死锁 | 用 Redisson 的 `RLock`（lua 脚本保证原子性）；或 `SET key value NX EX timeout` 原子命令 | 2026-06-18 |
| 83 | Redis | Redis 分布式锁 TTL 过短——业务执行时间超过锁 TTL，锁自动释放后其他线程进入，数据被覆盖 | 锁 TTL 设为业务预估最大时间的 2-3 倍；或看门狗自动续期（Redisson 默认续期） | 2026-06-18 |
| 84 | Redis | Redis 分布式锁主从切换丢锁——主节点加锁后数据未同步到从节点，主挂了从变主，锁丢失 | 用 Redisson 的红锁（RedLock）——多数节点加锁成功才算成功；或用 Zookeeper | 2026-06-18 |
| 85 | Redis | Redis 集群/哨兵模式下 Slot 迁移时服务不可用——运维操作导致大量 Key 找不到节点 | 集群变更前评估影响；客户端配置重试；变更窗口期停止写入 | 2026-06-18 |
| 86 | Redis | Redis 管道批量发送未限制大小——`pipeline.execute()` 一次发送过多命令，内存爆炸 | 批量命令分批执行（如每批 1000 条）；监控 pipeline 批次大小 | 2026-06-18 |

## 分布式锁陷阱

| # | 类别 | 陷阱描述 | 如何避免 | 发现 |
|---|------|---------|---------|------|
| 87 | 分布式锁 | 多服务实例用本地锁——本地 `synchronized` / `ReentrantLock` 只能锁单 JVM，多实例形同虚设 | 所有涉及多实例并发的场景必须用分布式锁（Redisson/Zookeeper/Nacos）；本地锁仅限单实例内 | 2026-06-18 |
| 88 | 分布式锁 | 分布式锁自动续期缺失——业务还没执行完锁就过期了，其他线程进入，数据被覆盖 | 用 Redisson 看门狗机制自动续期；或手动在业务执行中途续期；锁 TTL 留足余量 | 2026-06-18 |
| 89 | 分布式锁 | 分布式锁释放不当——释放锁时未校验 owner，导致释放了别人的锁 | 释放锁时必须验证当前 value（UUID）是否为当前线程持有；Redisson 自动处理 | 2026-06-18 |
| 90 | 分布式锁 | 分布式锁与本地缓存混用——加锁后写本地缓存，多实例各有一份缓存，数据不一致 | 分布式锁保护的临界区内的所有状态变更必须都是分布式的；禁止本地缓存 | 2026-06-18 |
| 91 | 分布式锁 | 分布式锁超时导致并发——锁 TTL 固定，业务执行时间波动大，到期自动释放并发进入 | 看门狗续期 + 业务幂等设计；业务代码必须幂等，锁只是优化不是保证 | 2026-06-18 |
| 92 | 分布式锁 | 数据库唯一索引替代分布式锁——唯一索引冲突抛异常处理慢，高并发时 DB 压力过大 | 唯一索引是兜底不是主力；高并发场景用 Redisson 锁；DB 唯一索引只做最终校验 | 2026-06-18 |
| 93 | 分布式锁 | 分布式锁死锁——加锁后抛异常没有 finally 释放锁，或在 finally 之前 return | 必须用 `try-finally` 包裹；建议用 Redisson 的 `tryLock` 自动释放 | 2026-06-18 |
| 94 | 分布式锁 | 分布式锁降级缺失——Redis 挂了锁不可用时系统直接崩溃，没有降级方案 | Redis 不可用时降级为本地锁 + 告警；或降级为不锁 + 业务限流 | 2026-06-18 |

## SQL 优化陷阱

| # | 类别 | 陷阱描述 | 如何避免 | 发现 |
|---|------|---------|---------|------|
| 95 | SQL | 隐式类型转换导致索引失效——字段是 varchar 但传了数字，MySQL 自动转换后不走索引 | 参数类型和字段类型必须一致；字段加引号确保字符串比较 | 2026-06-18 |
| 96 | SQL | OR 条件导致索引失效——`WHERE a = 1 OR b = 2`，OR 两边不是同一索引 | 拆为 UNION；或用 `IN`；确保 OR 的每个条件都能走索引或建联合索引 | 2026-06-18 |
| 97 | SQL | 联合索引顺序错误——建了 `(a,b,c)` 但查询只用 `(c,b)`，索引无法使用 | 联合索引遵守最左前缀原则；按等值查询频率排字段顺序；范围查询字段放最后 | 2026-06-18 |
| 98 | SQL | 函数/运算导致索引失效——`WHERE DATE(create_time) = '2026-01-01'` 不走索引 | 改为范围查询 `WHERE create_time >= '2026-01-01' AND create_time < '2026-01-02'` | 2026-06-18 |
| 99 | SQL | LIKE 前缀通配符导致全表扫描——`WHERE name LIKE '%xxx%'` 无法用索引 | 全文检索用 ES/MySQL 全文索引；前缀匹配 `LIKE 'xxx%'` 可以用索引 | 2026-06-18 |
| 100 | SQL | 多表 JOIN 数据量过大——未限制 JOIN 结果集大小，连表后行数爆炸 | JOIN 前各自过滤；用小表驱动大表；JOIN 结果必须带 LIMIT；禁止三表以上 JOIN | 2026-06-18 |
| 101 | SQL | ORDER BY 索引和 WHERE 索引不一致——ORDER BY 字段不在 WHERE 条件的索引中，MySQL 用 filesort | 确保 ORDER BY 字段在联合索引中且与 WHERE 等值条件一致；分析 `EXPLAIN` 结果 | 2026-06-18 |
| 102 | SQL | COUNT(*) 全表扫描——表数据量大时 `COUNT(*)` 极慢 | 用 `WHERE id > lastId` 计数；或维护统计表；前端分页总数用 `≈` 近似值 | 2026-06-18 |
| 103 | SQL | 事务过大——在一个大事务里做大量操作，锁持有时间过长 | 缩小事务范围；批量操作分段提交；读写分离读不走事务 | 2026-06-18 |
| 104 | SQL | 死锁——两个事务互相等待对方持有的锁，MySQL 自动检测后一个回滚 | 按固定顺序访问资源；减少事务内锁持有时间；大事务拆小 | 2026-06-18 |
| 105 | SQL | 不带条件的 DELETE/UPDATE——`DELETE FROM orders` 整表删除 | 必须带 WHERE 条件；大表删除用分批删除 + 暂停；先备份再执行 | 2026-06-18 |

## WebSocket 陷阱

| # | 类别 | 陷阱描述 | 如何避免 | 发现 |
|---|------|---------|---------|------|
| 106 | WebSocket | 单机内存爆炸——每连接存一个用户对象，万人在线时内存打满 | 轻量级连接管理；用户状态存 Redis；连接对象只存必要字段；监控连接数 | 2026-06-18 |
| 107 | WebSocket | 多实例消息不通——服务 A 的用户发消息，服务 B 的用户收不到 | 消息通过 Redis Pub/Sub 或 MQ 广播；或用 Redis Session 同步；网关层做消息路由 | 2026-06-18 |

| 109 | WebSocket | 断线重连风暴——大量客户端同时断线重连，服务被打爆 | 客户端加退避重连（exponential backoff）；服务端限流；CDN/WAF 做流量削峰 | 2026-06-18 |
| 110 | WebSocket | 消息顺序错乱——高并发时后发先到，业务逻辑依赖顺序导致数据错误 | 消息加序号 seq；消费端按序号排序；或用消息队列保证顺序 | 2026-06-18 |
| 111 | WebSocket | 认证信息过期未处理——JWT Token 过期后 WebSocket 连接还在，不知道用户已登出 | WebSocket 握手时校验 Token；连接中定期检查 Token 有效性；Token 过期主动关闭连接 | 2026-06-18 |
| 112 | WebSocket | 消息推送无幂等——网络重试导致消息重复推送，前端展示重复数据 | 消息加唯一 ID；前端根据 ID 去重；或用 `seq` 窗口去重 | 2026-06-18 |
| 113 | WebSocket | 大消息阻塞——推送大文件或大批数据时阻塞了其他消息的发送 | 消息大小限制（如单条不超过 64KB）；大文件走 CDN/OSS；大批量分片推送 | 2026-06-18 |
| 114 | WebSocket | 连接关闭时资源泄漏——用户断网/刷新页面时服务端连接未清理，线程/内存资源泄漏 | `@OnClose` 注解清理方法必须执行；心跳超时检测；Redis 维护连接状态 | 2026-06-18 |
| 115 | WebSocket | 恶意攻击——WebSocket 无同源限制，可被 CSRF 滥用做代理或 DoS | WebSocket 握手时校验 Origin；认证 Token 校验；连接数限流；监控异常流量 | 2026-06-18 |
| 116 | WebSocket | 订阅未授权——用户订阅了自己无权访问的频道（如管理员通知） | 订阅前校验权限；消息路由按租户+角色过滤；服务端只推送用户有权限的消息 | 2026-06-18 |
