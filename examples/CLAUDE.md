## 核心规则

### 1. 代码组织
- 小文件优先，单文件 200-400 行，最多 800 行
- 按功能/领域组织，非按类型
- 高内聚，低耦合

### 2. 代码风格
- 代码和注释中不使用 emoji
- 优先不可变数据结构
- 生产代码禁止 console.log
- 完善的错误处理

### 3. 测试
- TDD：先写测试
- 覆盖率 ≥ 80%
- 工具函数单元测试
- API 集成测试
- 关键流程 E2E 测试

### 4. 安全
- 禁止硬编码密钥
- 环境变量管理敏感数据
- 参数化查询
- 输入验证

## 文件结构

```
src/
├── app/              # 应用入口
├── components/       # 可复用组件
├── services/         # 业务逻辑
├── utils/            # 工具函数
└── types/            # 类型定义
```

## 可用命令

| 命令 | 说明 |
|------|------|
| `/code` | 进入代码开发模式 |
| `/review` | 代码审查 |
| `/test` | TDD 模式 |
| `/debug` | 调试模式 |
| `/plan` | 写实现计划 |
| `/verify` | 尾盘验证 |

## Git 工作流

- 约定式提交：`feat:`、`fix:`、`refactor:`、`docs:`、`test:`
- 不直接提交到 main
- PR 需要审查
- 合并前所有测试必须通过
