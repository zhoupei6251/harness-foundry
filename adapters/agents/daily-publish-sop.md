# Harness Kit 日常使用 SOP

> Harness Kit 标准化作业流程。按此执行，高效完成开发任务。
> 最后更新：2026-06-24

---

## 每日工作流程

### 上午：产出（约 30 分钟人类操作 + AI 自动执行）

#### 步骤 1：确认上下文
对 Harness Kit 说：
```
"继续上次的工作"
```
→ Harness Kit 自动读 MEMORY.md，恢复上下文

#### 步骤 2：执行任务
根据需求选择路由：
```
"设计方案" → design 路由
"写计划" → plan 路由
"开始实现" → implement 路由（拆 WU 并行）
"修 bug" → Leader 直做
```

#### 步骤 3：人工抽检
- 快速浏览生成的代码
- 检查是否符合预期
- 确认无明显问题

#### 步骤 4：保存进度
对 Harness Kit 说：
```
"今天就到这"
```
→ Harness Kit 更新 MEMORY.md、同步状态

## 新功能开发 SOP

### 第 1 步：设计方案
```
"设计 XX 功能"
```
→ brainstorming → 产出 spec → 暂停等确认

### 第 2 步：写计划
```
"写实现计划"
```
→ writing-plans → 产出 plan → 暂停等确认

### 第 3 步：开始实现
```
"开始实现"
```
→ dispatcher 并行产出

### 第 4 步：测试
```
"测试"
```
→ test-engineer 执行测试

### 第 5 步：审查
```
"代码审查"
```
→ reviewer 五轴审查

---

## 每周检查 SOP

### 周一：看进度
- 检查待办任务列表
- 评估当前 sprint 进度

### 周三：调整节奏
- 如果有阻塞项 → 优先解决
- 如果进度滞后 → 调整计划

### 周五：整理
- 记录本周遇到的问题
- 更新项目记忆
- 整理代码审查反馈

---

## 紧急情况处理

### 实现不通过测试
- coder 返修，最多 2 次
- 2 次仍不通过 → 人工介入

### 代码冲突
- 审查冲突文件
- 人工确认合并策略

### 架构问题
```
"重新设计 XX 模块"
```
→ brainstorming + planner 联合处理

---

## 多项目并行 SOP

同时开多个项目时：

```
上午：A 项目 2 个 WU（约 20 分钟）
中午：B 项目 1 个 WU（约 15 分钟）
晚上：审查 A 项目代码
```

每个项目独立一个 Git 分支，Harness Kit 通过上下文区分。
