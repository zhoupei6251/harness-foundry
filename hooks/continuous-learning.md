# 持续学习 Hooks

会话结束时自动提取模式、陷阱和经验，持续优化知识库。

## 触发时机

**Stop Hook**：每次响应结束后触发

## 提取内容

### 1. 代码模式（Code Patterns）
- 新发现的最佳实践
- 重复出现的代码结构
- 性能优化技巧
- 架构设计决策

### 2. 陷阱（Traps）
- 新发现的 bug 模式
- 常见错误
- 反模式（Anti-patterns）
- 安全漏洞

### 3. 经验（Lessons）
- 调试经验
- 重构经验
- 测试经验
- 协作经验

## 存储位置

```
references/
├── learned-patterns.md    # 学习到的模式
├── learned-traps.md       # 学习到的陷阱
└── lessons-learned.md     # 经验总结
```

## 提取规则

### 代码模式提取
```
IF 响应中包含:
  - 新函数/类/模块
  - 性能优化
  - 架构决策
THEN:
  - 提取核心逻辑
  - 记录适用场景
  - 存入 references/learned-patterns.md
```

### 陷阱提取
```
IF 响应中包含:
  - Bug 修复
  - 错误处理
  - 安全修复
THEN:
  - 提取问题描述
  - 记录解决方案
  - 存入 references/learned-traps.md
```

### 经验提取
```
IF 响应中包含:
  - 调试过程
  - 重构决策
  - 测试策略
THEN:
  - 提取关键洞察
  - 记录适用条件
  - 存入 references/lessons-learned.md
```

## 格式规范

### 模式格式
```markdown
## [模式名称]

**场景**: [何时使用]
**问题**: [解决什么问题]
**方案**: [具体实现]
**示例**: [代码示例]
**来源**: [会话日期]
```

### 陷阱格式
```markdown
## [陷阱名称]

**症状**: [表现形式]
**原因**: [根本原因]
**解决**: [修复方法]
**预防**: [如何避免]
**来源**: [会话日期]
```

### 经验格式
```markdown
## [经验标题]

**背景**: [什么情况下]
**洞察**: [关键发现]
**应用**: [如何使用]
**来源**: [会话日期]
```

## 质量控制

- 去重：检查是否已存在相似内容
- 验证：确保提取的内容准确有用
- 分类：按技术栈/领域分类
- 精简：去除冗余信息

## 使用方式

学习到的内容会在以下场景自动加载：

1. **写代码前**：加载 `references/learned-patterns.md`
2. **调试时**：加载 `references/learned-traps.md`
3. **Code Review**：加载 `references/lessons-learned.md`
